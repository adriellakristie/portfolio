OnTheWay

Mapbox GL JS Vanilla JS
Personal Project · 2026 Route-Aware Food Finder Visit Live Site
THE PROBLEM

"Food near me" tools only ever look at where you're standing right now — ignoring the fact that you're already headed somewhere, and the best stop might be a few minutes off your path, not next door.

THE SOLUTION

OnTheWay flips the question from "what's near me" to "what's along my way" — enter a start and destination, and it surfaces food options on your route, with the actual detour cost in minutes for each one.

How It Works

The whole flow runs through three Mapbox APIs working together — geocoding, directions, and search — wired into a small set of focused JS modules.

1
Geocode — the starting point and destination addresses are converted into coordinates via the Mapbox Geocoding API.
2
Route — the Mapbox Directions API calculates the optimal path between the two points and renders it on the map.
3
Search — restaurants near the route are pulled in and plotted as markers alongside the path.
4
Compare — each restaurant is scored by detour time, so you can see exactly how many extra minutes a stop costs before committing.
Detour Cost = Time(Start → Restaurant → Destination) − Time(Start → Destination, direct)

Code Organization

The app is split into focused modules so map logic, API calls, and UI stay decoupled: config.js centralizes API keys, mapbox-api.js handles geocoding, directions, and search, map-manager.js owns routes and markers, ui-manager.js drives the sidebar and status messages, and app.js ties the flow together.

Challenges Overcome

Calculating accurate detour time rather than straight-line distance to each restaurant
Searching for restaurants along an entire route corridor instead of a single point
Keeping the map responsive while juggling geocoding, directions, and search calls in sequence
